Introduction

What is the essential story being told by your site and what type of structure did you choose to implement.

FNH (Food Nutrition and Health) is a website that provides information about food and nutrition. It houses information accured from different authors with knowledge of food and nutrition. Anyone visiting the website can sign up and sign in, there details are stored in the browser database using localforage library. The localforage library was choosen because it automatically support different browsers and devices by internally using either indexedDB, websql or localstorage to store the data. This made the website much more dynamic. Signed in user can drop a review about the site at the homepage (which will appear automatically on the homepage review carousel), they can choose to become an author and create a new article at the blog page. The blog page entails a list of all the articles created by all the authors on the platform.

Accessibility

State 3 ways in which your site is accessible.

  1. The site was made accessible by ensuring that all img tags are accompanied by a descriptive alt attribute so that users using screen readers can read the image.

  2. The site was made accessible by ensuring that semantic tags are used to describe the content of the page.

  3. The site was checked for accessibility by using the WCAG 2.0 standards, which is a set of accessibility guidelines that are designed to meet the needs of the Web.

Usability

State 3 ways in which you have considered usability in your site.

  1. I asked friends and family to test the website and some provided feedback such as the font (too big). I modify the website based on all the feedback received, to make the website more user friendly.

  2. According to Nielson's Usability Principle (in references), the number 1 priciple is to ensure the visibility of system status. I followed this by ensuring the state of user authentication (either logged in or logged out) is visible at the navigation side. If they are signed in, their name displays on the navigation bar.

  3. The website has aesthetic and minimalist design. And it also allows user to recognize and recover from errors by displaying an alert for error and success message in order to keep the user informed. The website also has a loader to show the user that the website is loading due to it's asynchronous nature.

Learning

State 3 things you had to learn/find out by yourself when creating your site, and describe how you did that .

  1. I had to learn how to use the localforage library to store the data in the browser database using one of indexedDB, web sql or local storage. I came across the localforage library when reading documentation of indexdb on the MDN website (https://developer.mozilla.org/en-US/docs/Web/API/IndexedDB_API#see_also). I learnt through the docs that indexdb is not supported in Safari and other mobile browsers. Other alternatives are web sql and local storage. Luckily localforage automatically decides which storage to use based on the browser. Since i want to make my website accessible to everyone, i decided to use localforage and i learnt it via their documentation.(see references)

  2. I learnt about promises and how to handle asynchronous requests while using it with the try and catch pattern of exception handling. In the module.js file, i dynamically load the javascript file required for the present page a user is on. Through this also, i learnt how to organise scripts in the right order of dependencies. For instance, most variables and functions declared in the module.js file are used in other javascript files that follows the module.js file. I googled alot before i found an article that explains these concepts (https://developer.mozilla.org/en-US/docs/Web/HTML/Element/script).

  3. I learnt how to use some browser API (Application Programming Interface) like file reader for converting files to base64 encoded string. In the article.html page i learnt how to serve enitrely different content on the same page. I firstly tried to use document.write as gotten from https://www.w3schools.com/jsref/met_doc_write.asp, but this did not work. Then i manually used the innerHTML property of the document and style applied to the head for the same purpose.

What aspect(s) of your work do you think worked well and why?

The aspect of authentication (sign in and sign out) and dynamic changing of the site content worked well because it saves all the user details into the browser database, and the mechanism for the dynamic content is that: If the page needs to display an updated details or information, the whole page reloads. The system has been designed to access the data from the browser database while the website is being loaded, since accessing the database is asynchronous, a loader is displayed while the data is being loaded. Likewise, the article page is a dependent page of the blog page. For example: article.html?0 will load article at index 0 of the blog. But when user manually request for an articel that's not in the database (for example article.html?10), the page will automatically returns with Article Not Found page dynamically. This works because of the javascript used to check if the article exists in the database.

What aspect(s) of your work could be improved, and how might you do things differently another time?

The aspect of submitted review and articles can be improved by allowing users to delete and modify their own content that is on the website. Also user can be allowed to comment on the article page.
